## 一句话增量

**Before**：连续值预测依赖固定的参数分布假设（高斯混合、分位数回归等），模型表达能力受限于预设分布形式。
**After**：将连续值表示为有序量化码的累加和，通过从粗到细的递归预测自适应地捕捉数据分布，无需事先假设分布形态。

## 缺口分析

### 已有研究走到哪

| 方法类型 | 代表工作 | 局限 |
|---------|---------|------|
| 均方误差回归 | Wide&Deep、DIN等 | 假设数据服从均匀分布，无法建模长尾和复杂分布 |
| 分位数回归 | QR-MDN等 | 离散的分位点预测，全覆盖需要大量模型组合 |
| 混合密度网络 | MDN、MixtureNet | 依赖预设的高斯/拉普拉斯假设，参数调优困难 |
| 序列生成模型 | Transformer+回归头 | 仍是单点估计，未显式建模分布复杂性 |

### 本文填哪条缝

```
已有方法：假设一个分布形式 → 用模型拟合参数
     ↓
本文方法：不做假设 → 用量化码自适应划分数值空间
```

**核心假设**：
1. 任何连续值可以分解为有限个量化层级的和
2. 量化误差在各层级递减（ diminishing quantization errors）
3. 量化码之间存在序数结构（ordinal structure）

### 博导视角的关键问题

> **假设是否成立？** 残差量化在信号压缩领域有效，但在推荐系统回归任务中，"任意连续值可分解为量化码和"的假设需要验证——尤其是对于极端值（长尾）和边界情况。

## 白话方法

### 类比：猜数字游戏

想象玩一个猜数字游戏，要猜一件商品的价格。

**老方法**：
- 假设价格服从正态分布，猜"最可能"的值
- 问题：实际价格可能是幂律分布（如秒杀品），猜不准

**本文方法（RQ）**：
- 把价格拆成「百位 + 十位 + 个位」三级
- 第一级猜百位（粗粒度）：60%概率是1-3百
- 第二级基于第一级猜十位（细粒度）：如果百位是2，十位可能是0-9
- 第三级猜个位
- 最终价格 = 百位贡献 + 十位贡献 + 个位贡献

**核心思想**：不假设价格服从什么分布，而是用"量化层级"自动学习数据的真实分布结构。

## 关键概念

### 1. 残差量化（Residual Quantization）

**费曼式讲解**：想象你要表示0到100之间的任意数字，但你只能用有限个"积木"：
- 第1层积木：每块代表10（可用0-10块）
- 第2层积木：每块代表1（可用0-9块）
- 数字82 = 8个"10块" + 2个"1块"

残差量化就是这个思想的推广：先预测粗粒度值（剩余误差大），再逐层细化（剩余误差递减）。

**具体例子**：GMV预测中
- 第1层：预测量级（百/千/万）
- 第2层：预测百位
- 第3层：预测十位
- 最终 = 各层预测之和

### 2. 序数结构对齐（Ordinal Structure Alignment）

**费曼式讲解**：量化码之间不是独立的，而是有大小关系的。
- 如果码A代表"10-20"，码B代表"20-30"，码C代表"30-40"
- 那么 A < B < C 这个序数关系应该在学习到的嵌入空间中也体现出来

**实现方式**：引入对比学习/序数回归损失，让相似数值的码在嵌入空间中距离更近。

### 3. 递归预测（Recursive Prediction）

**费曼式讲解**：每层预测都基于前面所有层的预测结果进行调整。
- 第1层预测 → 得到粗略估计
- 第2层基于"第1层的误差"再预测 → 精细化
- 类似于"先定位城市，再定位街道，再定位门牌号"

## 研究启发

### 迁移

**问：RQL（残差量化）思想能否迁移到其他连续预测任务？**

答：完全可以。推荐系统中的连续值预测任务很多：
- 用户活跃天数预测
- 内容生命周期预测
- 广告出价优化
- 用户流失概率（虽然是概率，但可映射到[0,1]区间）

核心前提：目标值有自然的序数结构。

### 混搭

**问：能否与LLM增强方法结合？**

答：有趣的方向！
- 用LLM理解商品描述 → 提取隐式特征 → RQL预测GMV
- 用LLM做用户意图理解 → 增强序列编码 → RQL做价值预测
- 甚至可以探索：用Code-LLM自动生成量化层级配置

### 反转

**问：如果把预测改成"生成"呢？**

答：这是个好问题。本文是回归任务，但如果目标是：
- 生成"合理的价格区间"
- 生成"可能的GMV分布"

那RQL可以扩展为：预测量化码序列 → 从码序列采样 → 解码为连续值
这本质上是把VQ-VAE的思路引入推荐系统的连续值生成。
