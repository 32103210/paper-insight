# 论文解析报告

## Relative Advantage Debiasing for Watch-Time Prediction in Short-Video Recommendation

---

## 1. 一句话增量

**Before**: 直接用原始观看时间作为用户偏好的监督信号训练推荐模型，混淆因素（视频时长、流行度、用户行为习惯）会扭曲偏好估计。

**After**: 引入"相对优势"机制，将原始观看时间转换为条件分位数偏好的偏好信号，两阶段架构显式分离分布估计与偏好学习，配合分布嵌入实现高效参数化。

---

## 2. 缺口分析

### 已有研究走到哪

| 方向 | 现状 |
|------|------|
| **观看时间建模** | 通常将原始观看时间作为回归目标，忽视混淆因素 |
| **曝光偏差/选择偏差** | 主要关注Item的曝光概率，忽视用户行为的异质性 |
| **因果推断去偏** | 多聚焦于反事实推断，未专门处理时间信号的混淆问题 |
| **分位数回归** | 用于分布预测，但未结合推荐场景的因果分析 |

### 这篇论文填哪条缝

**缺口**：短视频推荐中，观看时间作为核心指标，却被视频时长和用户习惯等混淆变量严重扭曲，导致模型学到的是"短视频容易被看完"而非"用户真正喜欢"。

**填补**：提出条件分位数去偏框架，将观看时间转换为用户相对于同类用户群体的偏好程度（相对优势），切断混淆因素的传导路径。

### 核心假设

1. **条件独立性假设**：给定用户群体 $g_u$ 和视频群体 $g_v$，观看时间的条件分布可以分解
2. **群体同质性假设**：同群体内的用户/视频具有相似的观看行为模式
3. **单调性假设**：观看时间越长，偏好程度越高（该假设在去偏后更合理）

## 4. 白话方法讲解

### 日常类比：考试成绩的"相对优势"

想象一个老师想通过考试分数了解学生对知识的真实掌握程度：

**问题场景**：
- 张三得90分，李四得80分
- 但张三的试卷简单（全是选择题），李四的试卷难（有大量主观题）
- 直接比较分数不公平

**论文的方法**：
1. **建立参考分布**：老师先统计"简单试卷班级"和"难题试卷班级"各自的分数分布
2. **计算相对位置**：张三的90分在"简单试卷"班级排前20%，李四的80分在"难题班级"排前10%
3. **转换后的信号**：张三的偏好=0.2，李四的偏好=0.1 → 反映真实相对优势

**对应到论文**：
- 用户 = 学生
- 视频时长/流行度 = 试卷难度
- 原始观看时间 = 原始分数
- 条件分位数 = 相对优势分数

### 为什么这样更好？

| 传统方法的问题 | 本文方法的改进 |
|---------------|---------------|
| 短视频容易被完整看完 → 模型误以为用户喜欢短视频 | 完整看完短视频在"短视频群体"中很常见 → 相对优势低 |
| 热门视频曝光多、观看时间长 → 模型过度推荐热门 | 热门视频的观看时间参考分布高 → 相对优势被校准 |

## 5. 关键概念讲解

### 概念一：相对优势 (Relative Advantage)

**定义**：用户 $u$ 对视频 $v$ 的观看时间 $w$ 在条件分布 $P(w|g_u, g_v)$ 中的分位数位置。

$$RA(u,v) = F_{g_u,g_v}(w_{u,v}) = P(W \leq w_{u,v} | g_u, g_v)$$

**费曼式讲解**：想象你参加一场马拉松：
- 如果你跑了 2 小时，在"专业选手组"里可能是垫底（RA=0.1）
- 但在"业余爱好者组"里可能是冠军（RA=0.95）
- **相对优势**就是在"你所属的组别"里，你的表现超过了多少人

**具体例子**：
- 用户A看了一个10秒短视频，看完了（10秒）
- 用户B看了一个5分钟视频，看了50秒
- 原始数据：用户A观看时间(10s) < 用户B(50s)
- 去偏后：用户A在"短视频+快刷用户"分布中排90%分位，用户B在"长视频+深度用户"分布中排30%分位
- **结论**：用户A实际偏好更强

---

### 概念二：两阶段架构 (Two-Stage Architecture)

**核心思想**：将"估计参照分布"和"学习偏好"解耦

```
Stage 1: 离线/异步
┌────────────────────────────────────────────┐
│  用户分组 (g_u) × 视频分组 (g_v)            │
│         ↓                                   │
│  从历史数据估计条件分布 P(w|g_u, g_v)        │
│         ↓                                   │
│  生成相对优势信号 RA                         │
└────────────────────────────────────────────┘
                    ↓
                    ↓  查表或分布嵌入
                    ↓
Stage 2: 在线/实时
┌────────────────────────────────────────────┐
│  输入: 用户特征、视频特征、RA信号            │
│         ↓                                   │
│  模型预测: 用户对视频的真实偏好程度          │
│         ↓                                   │
│  排序/推荐                                  │
└────────────────────────────────────────────┘
```

**为什么这样做**：
- 分布估计不需要实时更新（用户群体和视频群体相对稳定）
- 偏好学习模型可以实时推理

---

### 概念三：分布嵌入 (Distributional Embedding)

**问题**：存储所有历史观看时间分布需要大量内存

**解决方案**：用神经网络"记住"分布

```
输入: 分位数 τ (0到1之间)
     ↓
┌────────────────────────┐
│   共享的 τ编码层         │  τ → 128维向量
└────────────────────────┘
         ↓
┌────────────────────────┐
│   群体嵌入层             │  (g_u, g_v) → 128维向量
└────────────────────────┘
         ↓
      concat
         ↓
┌────────────────────────┐
│   MLP 回归层             │  输出: Q(τ) = 预测的观看时间分位数值
└────────────────────────┘
```

**类比**：就像你不需要记住所有同学的成绩，只要记住"这个班平均分80，标准差10"就能推断任何分数的位置。分布嵌入就是让模型"记住"分布的参数。

## 8. 研究启发

### 迁移启发

**问：这个去偏思想可以迁移到其他指标吗？**

**答**：完全可以。

| 源指标 | 目标指标 | 迁移要点 |
|--------|---------|---------|
| 观看时间 | 点击率 (CTR) | 需要估计"参考点击率分布"，考虑位置偏差、流行度 |

**具体方向**：直播推荐中的"观看时长"去偏、短视频的"完播率"去偏（完播率本身已经是比例，但仍受视频长度影响）

1. **+ 因果推断工具**（Doubly Robust、IPS等）
   - 将相对优势作为反事实结果，用因果推断框架统一
   - 可能解决"分组边界模糊"的问题

2. **+ LLM/多模态**
   - 用视频内容理解（CLIP）作为分组依据，而非简单基于时长/类别
   - LLM 可以生成更细粒度的视频"类型描述"用于分组

3. **+ 多任务学习**
   - 同时预测相对优势和其他信号（点击、分享）
   - 相对优势作为辅助任务可能提升主任务泛化

### 反转启发

**问：如果反转核心思想会怎样？**

**答**：

**反转1：从"相对优势"到"绝对偏好"**
- 不做去偏，而是用因果图显式建模混淆因素的贡献
- 可能更精确但需要更多假设

**反转2：从"两阶段"到"端到端"**
- 让模型端到端学习"哪些特征应该被去偏"
- 牺牲可解释性换取灵活性

**反转3：从"用户视角"到"平台视角"**
- 不仅去偏用户偏好，还考虑平台目标（多样性、新鲜度）
- 可能需要多目标优化框架

**最有价值的反转**：
> 不做"去除混淆"，而是做"可控混淆"——让模型学习在什么场景下应该考虑时长因素，在什么场景下应该忽略。这可能比简单去偏更符合业务需求。
