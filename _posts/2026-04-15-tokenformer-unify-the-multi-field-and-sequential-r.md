## 缺口分析

### 已有研究走到哪

推荐系统领域长期存在两条独立发展路线：
- **特征交互派**：DNN、DeepFM、DIN 等，专注建模多字段类别特征的交叉关系
- **序列建模派**：DIN、DIEN、SASRec 等，专注从用户历史行为序列中捕捉兴趣漂移

近期研究开始尝试用统一架构（如 transformer-based）同时处理两类任务，但普遍存在**维度不匹配**的隐患——非序列字段（如用户画像）的 embedding 维度与序列字段的维度往往不一致。

### 这篇填哪条缝

本文的贡献是一个**诊断 + 解决方案**的组合拳：

| 层次 | 内容 |
|------|------|
| **诊断** | 首次命名并系统分析 Sequential Collapse Propagation (SCP) 现象——非序列字段的低质量信号污染序列特征的表达空间 |
| **方案** | TokenFormer 架构：BFTS 注意力 + NLIR 非线性变换 |

### 核心假设

- **假设 A**：非序列字段（如 [字段数 × 嵌入维度]）与序列字段（[序列长度 × 嵌入维度]）维度差异大，直接混合会导致"维度崩溃"
- **假设 B**：底层用 full attention 让两类 token 充分交互，上层用 sliding attention 收缩视野，可以缓解崩溃
- **假设 C**：引入单边非线性乘法变换可以增强维度鲁棒性

## 白话方法

### 类比：宴会桌与连续剧

想象你在组织一场**推荐派对**，有两种客人：

| 类型 | 多字段用户（卡片人） | 序列用户（电影人） |
|------|---------------------|-------------------|
| 特征 | 一张信息卡：性别、职业、地点等 | 一部连续剧：看过的一系列商品 |
| 维度感 | 卡片是"死的"，信息固定 | 电影是"活的"，有时序和情节 |

**统一建模的难题**：把卡片人和电影人放在同一个大圆桌上聊天，但如果他们"说话的维度"不一样——比如卡片人用二维词描述，电影人用高维情节叙事——直接混合会导致电影人的思维崩溃成二维碎片（SCP）。

**TokenFormer 的解决方案**：

1. **BFTS 策略**：宴会分两阶段
   - **前半场**（底层）：所有人自由交流，充分互动，卡片人和电影人互相认识
   - **后半场**（上层）：电影人缩回小圈子，只跟最近看过的东西聊，不让卡片人的碎片思维继续污染

2. **NLIR 机制**：给电影人的对话加"过滤网"
   - 普通的注意力让所有输入无差别交互
   - NLIR 在 Query 端加了个非线性门控：电影人可以主动"屏蔽"来自卡片人的低质量信号，只保留核心内容

## 关键概念

### 概念 1：Sequential Collapse Propagation (SCP)

**什么是它**：当非序列字段（如用户画像）与序列字段（行为序列）统一建模时，非序列字段的"维度差信号"会像病毒一样传播给序列特征，导致序列表达空间坍缩。

**具体例子**：
```
假设：
- 卡片特征维度: d_f = 16 (低维)
- 序列特征维度: d_s = 64 (高维)

直接混合后：
序列特征的某个维度 d_s[i] 接收到大量 d_f 级别的信号 → 
该维度被"污染" → 整体表达空间坍缩到 d_f 级别 → 序列信息丢失
```

**为什么重要**：这是首次被系统命名的问题，解释了为什么之前的统一尝试要么效果不好，要么需要大量人工调参。

### 概念 2：Bottom-Full-Top-Sliding (BFTS)

**什么是它**：一种分层注意力策略——底层用全注意力让异质 token 充分交互，上层用收缩窗口滑动注意力限制信息传播范围。

**日常类比**：
- 底层全注意力 = 新生开学典礼，所有人互相打招呼
- 上层滑动注意力 = 分成小组讨论，只跟周围人聊

**关键洞察**：这样设计是因为"充分互动"和"防止污染"是一对矛盾，用分层策略化解。

### 概念 3：Non-Linear Interaction Representation (NLIR)

**什么是它**：在注意力计算时，对 Query 做非线性变换 + 逐元素乘法门控：Q = σ(W_Q · x) ⊙ x，仅在 Query 侧施加非线性，保持 K/V 线性。

**与标准 attention 的对比**：

| 操作 | 标准 Attention | NLIR |
|------|---------------|------|
| Q 变换 | 线性：W_Q · x | 非线性+门控：σ(W_Q · x) ⊙ x |
| K 变换 | 线性：W_K · x | 线性 |
| V 变换 | 线性：W_V · x | 线性 |
| 效果 | 无差别交互 | 自适应过滤 |

**为什么只做单边**：避免过度非线性导致训练不稳定，同时单边已经足够产生"维度筛选"效果。

## 研究启发

### 迁移

**问 1：这篇工作的思路能否迁移到其他领域？**

可以。BFTS 的"分层注意力 + 视野收缩"思路可以迁移到：
- **多模态学习**：图像 token 和文本 token 的维度差异问题
- **知识图谱**：实体描述（字段型）和路径（序列型）的统一建模
- **时序预测**：静态特征和时序特征统一处理

**问 2：SCP 问题是否在其他任务中也存在？**

很可能。凡是需要**异质 embedding 维度统一建模**的场景都可能触发 SCP，比如：
- 多语言 NLP 中，低资源语言和高资源语言的 embedding 维度不同
- 多任务学习中，不同任务的数据规模和维度差异

### 混搭

**问 3：能否与其他技术混搭组合？**

强烈推荐几个方向：

| 混搭方向 | 理由 |
|---------|------|
| + 因果推断 | SCP 本质上是一种混杂（confounding），可以用因果工具显式解耦 |
| + 对比学习 | 增强序列表征的判别性，与 NLIR 的维度鲁棒性形成互补 |
| + 大模型蒸馏 | TokenFormer 可以作为 teacher，蒸馏到轻量模型 |
| + 缩放定律 | 研究 BFTS/NLIR 在不同模型规模下的 scaling 行为 |

### 反转

**问 4：如果反转这篇论文的核心假设，会怎样？**

**反转假设**：不是"防止维度崩溃"，而是"主动利用维度差异"。

**可能的方向**：
- 设计一种机制，让低维字段和高维字段**刻意交叉**，而不是隔离
- 比如：低维字段作为"路由信号"，动态决定高维字段的注意力模式
- 这本质上是一种**条件计算**思想，可能比 BFTS 的固定分层更灵活

**反转问题定义**：不是统一两类模型，而是**保留各自优势，用轻量级融合层交互**。

- 两个独立的 encoder + 轻量 cross-attention
- 避免 SCP 的同时保持各自主干的完整性
- 代价是参数量增加，但可能更灵活
