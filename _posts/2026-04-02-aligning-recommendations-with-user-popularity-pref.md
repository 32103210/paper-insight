## 缺口分析

### 已有研究走到哪儿了

- **诊断层**：识别出流行度偏差的存在（Item-level popularity bias measurement）
- **治疗层**：在训练阶段通过对抗、去偏训练、因果推断等手段降低全局流行度偏差（Global debiasing）
- **核心盲点**：默认「热门 = 不该推」，忽视了用户本身就存在**流行度偏好差异**——有人天生爱追爆款，有人专门找冷门好东西

### 本文填了哪条缝

| 缺口 | 本文对策 |
|------|----------|
| 缺乏用户级别的偏差度量 | 提出 **Popularity Quantile Calibration**——衡量「你过去喜欢的流行度」vs「现在推给你的流行度」之间的校准误差 |
| 全局去偏一刀切 | 提出 **SPREE**——允许每个用户有独立的流行度偏差方向和强度，推理时自适应调整 |
| 训练时去偏成本高 | SPREE 是**推理时（inference-time）**方法，不需要重新训练模型 |

### 核心假设

1. 用户的浏览/点击历史中蕴含其「个人流行度偏好信号」
2. 这个信号可以在模型的表征空间中找到一个方向（direction）
3. 朝这个方向「推一把」就能让推荐结果对齐该用户的偏好

### SPREE 工作流程

```
用户历史行为
     │
     ▼
┌─────────────────────────────┐
│   个人流行度偏好估计器       │
│  (Preference Estimator)     │
│                             │
│  输入: 用户交互序列          │
│  输出: ρ ∈ [0,1]            │
│  (该用户有多爱热门内容)       │
└──────────────┬──────────────┘
               │
               ▼
    ┌──────────┴──────────┐
    │   流行度方向向量      │
    │   (Popularity Direction)│
    │   从预训练模型中      │
    │   用方向向量分析得到   │
    └──────────┬──────────┘
               │
               ▼
┌───────────────────────────────────────┐
│         激活空间引导 (Activation Steering) │
│                                       │
│  原始激活 h ──► h + λ·ρ·d             │
│                   ↑    ↑    ↑        │
│              标量  权重 方向向量        │
│              (自适应调整强度)           │
└───────────────────────┬───────────────┘
                        │
                        ▼
              输出层 + 最终推荐
```

### Popularity Quantile Calibration 计算逻辑

```
第一步：把物品按流行度排序，分成 Q 个分位桶（quantiles）

第二步：统计用户在每个桶中的交互比例
        P_user = [p_1, p_2, ..., p_Q]

第三步：统计推荐系统在每个桶中的推荐比例
        R_sys = [r_1, r_2, ..., r_Q]

第四步：计算 PQC 指标
        PQC = Σ |P_user[q] - R_sys[q]| / 2
        (归一化的 L1 距离，0=完美校准，1=完全错位)
```

## 白话方法

### 如果你是普通读者

想象你去一家唱片店：

> **老办法**：店员觉得所有人都不该买排行榜冠军，所以故意不给你推荐。你明明是个「Billboard 爱好者」，却被迫看一堆地下乐队的 CD。

> **SPREE 的做法**：店员先观察你以前买过什么——如果你买的都是爆款专辑，那就给你推爆款；如果你专挑小众独立音乐，就给你推小众。它不是「不准推热门」，而是「让你每次来都能买到你真正想要的」。

### 如果你要向工程师解释

SPREE = 推理时激活扰动 + 用户自适应权重 + 方向向量

- **方向向量怎么来**：对模型所有层的激活做主成分分析（PCA），第一主成分就对应「流行度方向」（热门 vs 冷门的编码方向）
- **权重怎么算**：训练一个轻量级偏好估计器，从用户历史序列预测该用户的 ρ 值
- **为什么说推理时**：模型权重不变，只是在前向传播时给激活向量加个偏移，不需要重新训练

## 关键概念

### 概念 1：Popularity Quantile Calibration

**费曼式讲解**：想象你去医院体检，报告说「你的身高分布」vs「你实际的身高」——如果报告说你应该在 90% 分位但你实际在 30% 分位，这就是「校准失败」。

PQC 就是这个思路：把物品按热度分成 10 档（每档 10% 的物品），看看你历史点击的「档次分布」和推荐给你的「档次分布」是否一致。

**具体例子**：
- 用户 A：点击的 80% 是 top 20% 热门 → 偏好热门
- 用户 B：点击的 80% 是 bottom 20% 冷门 → 偏好冷门
- 推荐系统：给两人都是 70% top 20% → 对用户 A 校准好，对用户 B 校准差

### 概念 2：Activation Steering（激活引导）

**费曼式讲解**：大语言模型里有个经典发现——模型内部某个神经元的激活值与「情绪」相关，把这个值调高，模型输出的语气就变了。激活引导就是在推理时「拧这个旋钮」，而不改变模型本身。

**具体例子**：
- 某模型中，第 37 层第 502 个神经元的激活值与「热门程度」高度相关
- 热门物品 → 该神经元激活值高
- SPREE 的做法：如果要推荐给「偏好冷门」的用户，就把这个值往下压一点

### 概念 3：Sequential Recommender（序列推荐）

**费曼式讲解**：推荐系统知道你最近看了 A → B → C，猜测你下一步可能想看 D。

SPREE 基于这个序列做推荐，但在输出之前，用激活引导「微调」结果，使其符合你的流行度偏好。

### 主流框架对比

| 维度 | 传统去偏方法（如 DCF、MACR） | 本文 SPREE |
|------|---------------------------|------------|
| **目标** | 降低全局流行度偏差 | 提升用户-推荐对齐度 |

| **粒度** | 全局统一 | 每个用户自适应 |

## 研究启发

### 迁移：这套「用户-推荐对齐」的思路能不能迁移到其他偏差？

| 偏差类型 | 迁移思路 |
|----------|----------|
| **曝光偏差（Exposure bias）** | 用类似的 Quantile Calibration 衡量「用户看到的 vs 实际喜欢的」 |
| **位置偏差（Position bias）** | 设计 Position-level Alignment，让排名靠后的好物品也能被对齐 |
| **价格敏感度偏差** | 电商场景中，用户对价格区间的偏好 vs 推荐商品的价格分布 |
| **多样性偏差** | 用户的「多样性偏好」vs 推荐列表的多样性程度 |

**关键问题**：每个偏差都需要找到其「方向向量」——流行度可以用 PCA，其他偏差呢？

---

### 混搭：能不能和其他方法组合？

**推荐 + LLM**：在 LLM-based 推荐系统中，用户偏好可以通过 prompt 注入，激活引导可以被「prompt steering」替代。

**推荐 + Reinforcement Learning**：SPREE 是静态的，但用户偏好会随时间变化——能否用 RL 让 ρ 值动态调整？

**推荐 + Knowledge Graph**：将物品的知识图谱嵌入融入方向向量，使流行度方向更有语义可解释性。

---

### 反转：如果把这个思路反过来想？

> **如果「不对齐」反而是好事呢？**

- 平台的商业目标可能是「推高热门内容的曝光」而不是「让用户开心」
- 新用户引导场景中，主动推荐热门可能比对齐历史偏好更合理（因为你没什么历史）
- **反向 SPREE**：故意推「偏离用户偏好」的推荐，用于探索用户潜在兴趣（Exploitation vs Exploration 的平衡）

**核心反转**：把「对齐」从目标变成一种**可调节的手段**，而不必总是最大化对齐。

## 总结

这篇论文的核心价值在于**重新定义问题**：流行度偏差不是「该不该推热门」，而是「推的是不是用户想要的」。在这个框架下，SPREE 提供了优雅的推理时解决方案，Popularity Quantile Calibration 提供了精准的度量工具。两者结合，为推荐系统的公平性研究开辟了一条新路。
