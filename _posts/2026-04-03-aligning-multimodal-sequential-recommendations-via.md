## 1. 一句话增量

**Before**: 多模态序列推荐系统使用确定性硬负样本进行DPO训练，容易因假负例产生错误的抑制梯度。

## 2. 缺口分析

### 已有研究走到哪
| 研究方向 | 进展 | 边界 |
|---------|------|------|
| LLM领域的DPO | 成熟的RLHF替代方案，理论上可处理偏好数据 | 需要显式偏好对（chosen/rejected） |
| 推荐系统的隐式反馈 | 广泛使用硬负样本策略 | 未区分"未观测"与"真负" |
| 多模态推荐 | 视觉+文本+行为序列融合 | 训练目标仍依赖传统pairwise loss |

### 本文填哪条缝

```
传统做法: 用户点击 ✓ → 正样本，未点击 ✗ → 负样本（粗暴假设）
     ↓
问题: 用户未点击 ≠ 用户不喜欢（可能是未曝光、位置偏置等）
     ↓
本文洞见: DPO在LLM中用显式偏好对，但在推荐中是隐式的"伪偏好"
     ↓
核心改进: 让负样本选择"软一点"、"随机一点"，减少假负例的误导
```

### 核心假设
1. **隐式反馈的不可靠假设**: 未观测物品不能简单视为负样本
2. **随机性的优化价值**: 确定性 → 随机性 带来梯度噪声但避免了方向错误
3. **动态候选池的有效性**: top-K池随训练动态更新，保证负样本质量

## 4. 白话方法

### 场景类比：餐厅推荐系统

想象你在帮朋友推荐餐厅：

**传统方法（确定性硬负）**:
> "你去过这家川菜馆觉得好吃，那旁边那家湘菜馆你没去过，我就假设你不喜欢湘菜，以后不推荐了。"

问题：人家只是没去过那家湘菜馆，不代表不喜欢啊！

**本文方法（随机采样软负）**:
> "你喜欢川菜，我找几家你没去过的湘菜馆、粤菜馆、火锅店，从里面随机挑几家来问：'这家和那家川菜馆比，你更想去哪家？' 如果被随机问到的人说'湘菜馆也行啊'，那我就知道之前假设错了。"

**为什么更好**:
1. 避免"一刀切"：不是所有湘菜馆都排除
2. 保留信息量：偶尔抽到"真不喜欢"的，反馈价值更大
3. 随机性防止过拟合：老问同样的问题会陷入偏见

## 5. 关键概念

### 概念一：DPO (Direct Preference Optimization)
| 维度 | 解释 |
|------|------|
| 本质 | 一种直接优化"偏好"而非"损失函数"的训练方式 |
| LLM起源 | GPT-4等模型用人类反馈微调，DPO是更简洁的替代方案 |
| 推荐场景 | 将"用户点击"视为chosen，"随机采样未点击"视为rejected |
| 公式含义 | 让正样本得分永远比负样本高，即使考虑了"随机扰动"后也是 |

**举例**: 如果用户点击了A，没点B，传统方法只管"A排第一"。DPO要求"A比B好"，而且"换一批B，A还是比B好"。

### 概念二：假负例 (False Negative)
| 维度 | 解释 |
|------|------|
| 定义 | 实际是正样本，但被错误标记为负样本 |
| 产生原因 | 用户没看到/没点 ≠ 不喜欢 |
| 危害 | 产生"抑制梯度"：模型被训练成"远离"这个物品 |
| 本文解法 | 随机采样降低每个负样本的"确定性"，减少假负例的影响 |

**举例**: 朋友说"这家餐厅我没去过"，不代表"我讨厌这家餐厅"。

### 概念三：Sparse MoE (稀疏专家混合)
| 维度 | 解释 |
|------|------|
| 传统 | 每个输入经过所有专家（计算量大） |
| MoE改进 | 只激活少数专家（门控网络决定） |
| Sparse含义 | 不是激活所有，而是激活最相关的Top-K |
| 本文用途 | 多模态特征（文本、图像、行为）由不同专家处理 |

**举例**: 餐厅评价来了三条：一条文字评论、一张食物照片、一条历史订单。MoE会让"文本专家"处理评论，"图像专家"处理照片，"行为专家"处理订单，但每次只激活最相关的2个专家。

## 6. Before vs After

| 维度 | 主流框架 | 本文框架 |
|------|---------|---------|
| **负样本来源** | 确定性硬负：固定策略（如in-batch采样） | 随机采样：从动态top-K候选池中随机选择 |
| **假负例处理** | 忽略：默认未观测=不喜欢 | 缓解：随机性降低单一样本的误导性 |
| **优化目标** | Pairwise/BPR loss：正 > 负 | DPO loss：正 > 负 + 对抗扰动鲁棒 |
| **训练稳定性** | 依赖负样本质量，容易梯度震荡 | 随机性带来平滑效果 |
| **计算开销** | 无额外开销 | 候选池维护 + 随机采样（可忽略） |
| **多模态融合** | 拼接或简单注意力 | Sparse MoE（可选，可插拔） |
| **推理成本** | 标准 | Nearly unchanged（MoE仅影响编码阶段） |

**核心转变**: 从"精确控制负样本"到"承认不确定性，用随机性对冲"

### 混搭启发
**Q: 能和哪些方向混搭？**

| 混搭方向 | 可能的化学反应 |
|---------|---------------|
| 因果推断 | 用因果图建模"曝光"vs"偏好"，减少假负例 |
| 对比学习 | DPO + 对比学习 = 既要正负分离，又要扰动鲁棒 |
| 扩散模型 | 用扩散生成"软负样本"替代随机采样 |
| LLM推荐 | 用LLM生成偏好解释，再用DPO微调 |

### 反转启发
**Q: 如果反过来做，会怎样？**

**反转思路**: 不把"随机性"当作负样本策略，而是把"确定性"当作DPO的正则项。

```
Original: 随机采样负样本 → 减少假负例影响
Reversed: 添加对抗性确定性负样本 → 增强模型判别能力
```

**意义**: 如果"确定性负样本"和"随机负样本"组合使用，可能同时获得：
- 随机性带来的鲁棒性
- 确定性负样本提供的高区分度梯度
