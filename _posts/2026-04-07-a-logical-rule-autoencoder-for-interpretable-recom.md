## 1. 一句话增量

**Before**: 推荐系统使用黑盒神经网络，通过隐向量做决策，用户看不懂、开发者也说不清为什么推荐了这个item。

## 2. 缺口分析

### 已有研究走到哪

| 研究路线 | 现状 | 局限 |
|---------|------|------|
| 深度学习推荐 (NCF, DMF等) | AUC/Recall 指标优秀 | 完全黑盒，决策不可解释 |
| 可解释推荐后处理 | LIME, SHAP 等 | 事后解释，非模型内置；解释可能不稳定 |
| 规则化推荐 | 人工设计规则 | 依赖专家知识，泛化能力差 |
| 神经符号推荐 | 少数探索 | 通常需要预定义规则模板 |

### 本文填哪条缝

**缺口**: 如何让推荐模型从数据中**自动学习**出人类可读的逻辑规则，同时保持竞争力？

**本文解法**: 在 Autoencoder 框架中，用**可学习的规则神经元**替代传统隐向量，每个神经元表达一条 IF-THEN 重建规则。

### 核心假设

1. 用户行为数据中蕴含可被逻辑规则捕捉的pattern
2. AND/OR 两种逻辑操作足以表达大多数协同过滤模式
3. 通过权重符号编码否定，在不增加维度的情况下实现功能完备性

## 4. 白话方法

### 日常类比：像一位"老店员"顾问

想象你走进一家书店，店员跟你说：

> "我给你推荐这本书，因为**看了《百年孤独》但没买《尤利西斯》的人**，大多会喜欢这本。"

这个推荐理由非常清晰：
- **条件是什么**: 买了A，没买C
- **结论是什么**: 推荐B
- **规则是什么**: AND（两个条件都要满足）

**LIA就是在自动学习这种规则**。传统深度学习模型像个神秘的书店——你不知道它为什么推荐，但它就是推荐了。LIA像个透明的老店员，每条推荐都附带可读的理由。

### 技术要点简化

| 组件 | 类比 |
|------|------|
| Autoencoder | 先"压缩"用户兴趣，再"解压"预测 |
| 规则神经元 | 一个"IF-THEN"规则盒子 |
| AND/OR Gate | 自动判断这条规则应该用"且"还是"或" |
| 权重符号 | 正数=需要有这个条件；负数=不能有这个条件 |

## 5. 关键概念

### 概念一：规则神经元 (Rule Neuron)

**什么是它**: 一个能表达"IF 条件 THEN 结论"的数据结构。

**工作原理**:
```
输入x (用户历史向量) → 规则神经元 → 输出r_k (规则激活强度)
```

每个规则神经元有：
- 一个权重向量 **w_k**（定义条件）
- 一个门参数 **g_k**（定义AND/OR）
- 一个偏置 **b_k**（定义阈值）

**具体例子**:
```
规则神经元 k:
  w_k = [+1.2, -0.8, +0.5, ...]  (对应物品A,C,D...)
  g_k = 0.3  → 倾向于 AND 操作
  b_k = 0.5
  
用户历史 x = [1, 0, 1, ...]  (买了A，没买C，买了D...)
→ r_k = σ(w_k · x + b_k) ≈ 激活了（因为A和D满足条件）
```

### 概念二：Gate 自动选择 AND/OR

**核心创新**: 让模型自己学应该用哪种逻辑。

**数学表达**:
```
AND_output = Π σ(w_ij · x_j)      (概率连乘)
OR_output  = 1 - Π (1 - σ(w_ij · x_j)) (德摩根律)

门控混合: r_k = g_k · OR + (1-g_k) · AND
```

**类比**:
- **AND规则**: "买过A **且** 买过B的人" → 要求两个条件都满足
- **OR规则**: "买过A **或** 买过B的人" → 满足任一条件即可
- **Gate g_k**: 自动学习这条规则更适合哪种模式

### 概念三：负权重编码否定

**问题**: 如何表达"没买过A"这种否定条件？

**传统做法**: 增加一个维度 A' = 1 - A（维度翻倍）

**LIA做法**: 直接用**负权重**
```
w_kj > 0:  x_j = 1 时贡献为正 (即"买过A")
w_kj < 0:  x_j = 1 时贡献为负 (即"买过A"反而降低激活)
           x_j = 0 时贡献为正 (即"没买过A")
```

**效率对比**:
| 方法 | 维度 | 参数数量 |
|------|------|---------|
| 传统扩展 | 2n | 多 |
| LIA负权重 | n | 少 |

**优点**: Autoencoder框架本身成熟；AND/OR门控设计巧妙；负权重编码否定是solid的工程 tricks。

**审慎点**:

## 8. 研究启发

### 迁移启发

**问**: LIA的逻辑规则层能否迁移到其他任务？

**答**: 完全可以。例如：
- **搜索排序**: 用逻辑规则替代GBDT，做可解释的搜索排序
- **金融风控**: 规则本身可审计，适合合规要求
- **医疗诊断**: AND/OR规则符合医学决策逻辑

### 混搭启发

**问**: 如果把LIA的规则层和LLM结合会怎样？

**答**: 可以形成"规则→自然语言解释"的pipeline：
```
规则层输出 → 提示模板 → LLM生成自然语言解释
```
这样既有规则的可验证性，又有LLM的解释流畅性。

### 反转启发

**问**: 如果反过来，用逻辑规则做**生成**而非**推荐**呢？

**答**: 很有趣的思路。例如：
- 生成"符合某类用户偏好的商品描述"
- 生成"满足特定规则的新闻推荐列表"
- 可控推荐：用户指定规则，系统生成符合规则的推荐
