## 缺口分析

### 已有研究走到哪

1. **传统优化范式**：业界普遍使用行为信号（CTR、观看时长）作为优化目标，这些是用户满意度的"代理指标"
2. **问卷应用局限**：少量研究尝试引入问卷数据，但多用于离线评估或周期性校准，无法实时反馈到模型
3. **信号融合困难**：问卷信号稀疏（约 0.1% 用户会填写），直接混合训练会被行为数据淹没
4. **多任务学习的表层应用**：虽有研究用多任务学习分离不同信号，但未针对"稀疏满意度信号"设计专门架构

### 这篇论文填哪条缝

本文填补了**稀疏满意度信号无法端到端实时注入推荐模型**这一缺口，提出：
- 多任务架构专门保护稀疏问卷信号不被淹没
- LoRA 模块实现参数隔离的满意度预注入
- DPO 优化目标实现实时在线学习

### 核心假设

| 假设 | 内容 |
|------|------|
| H1 | 问卷反馈能代表用户真实满意度（虽然样本少，但信号准） |

## 白话方法

### 类比讲解：让推荐系统学会"读心术"

想象你要教一个推销员了解顾客的**真实满意度**，而不是只看顾客是否掏钱买了东西。

**传统方法的问题**：
> 推销员只看"顾客买没买"、"买了多少钱"。但有些顾客可能买了之后后悔，有些看了半天觉得不满意就是不买。用这些间接信号判断满意度，就像只看分数评价学生——考高分的不一定学懂了。

**本文的做法**：

1. **单独开辟一条学习路径**：推销员既要卖货，也要定期问顾客"你真的满意吗？"——用多任务架构，把"问卷满意度"当成独立任务单独学，不让"卖货数据"把"满意度反馈"冲淡

2. **用小插件快速调整**：给推销员一个"微型备忘录"（LoRA），专门记录从问卷中学到的偏好，每次顾客反馈就更新这个备忘录，不影响他平时卖货的能力

3. **实时学习新偏好**：用一种叫 DPO 的方法，像"对弈学习"一样，模型同时看到"问卷说满意的推荐"和"问卷说不满意的推荐"，反复练习，直到能准确预测哪种推荐真正让用户开心

## 关键概念

### 概念 1：LoRA（Low-Rank Adaptation）

**费曼式讲解**：LoRA 就像给一个已经训练好的"老司机"加一个"新手小本本"。

- 老司机 = 预训练好的推荐模型（已经学会了很多推荐技能）
- 新手小本本 = LoRA 模块（一组小矩阵，专门记录新学的技能）

**具体例子**：
- 原本要更新模型全部参数 = 重新培训整个老司机
- 用 LoRA = 只更新"小本本"，老司机的技能不动
- 好处：更新快、不忘记原来的本事、需要的计算资源少

### 概念 2：DPO（Direct Preference Optimization）

**费曼式讲解**：DPO 就像"题海战术"中的"错题本学习"。

- 不是死记硬背标准答案
- 而是把"正确答案"和"错误答案"放一起对比学习
- 模型学会：哪种推荐更像问卷用户说"满意"的

**具体例子**：
- 输入：用户 u 看了一个视频 A（问卷说满意），看了视频 B（问卷说不满意）
- DPO 让模型学会：调整参数让 A 的得分高于 B
- 优点：不需要像传统监督学习那样"死记"正确答案，而是学会"偏好关系"

### 概念 3：多任务学习中的任务权重

**费曼式讲解**：多任务学习就像让一个学生同时学语文和数学。

- 问题：学生时间有限，如果语文作业太多，数学就练得少
- 解决：设计一个"权重分配器"，自动决定学生应该在每科花多少精力
- 本文的特殊处理：问卷任务非常稀疏（可能 1000 个行为数据才有 1 个问卷），需要特殊保护机制

## 研究启发

### 迁移启发

**问：这项技术可以迁移到其他领域吗？**

答：可以。

EASQ 的核心思想——"用独立参数路径保护稀疏信号 + 参数隔离注入 + 偏好对齐优化"——可以迁移到：
- **搜索系统**：用查询满意度问卷替代推荐满意度问卷
- **广告系统**：用户对广告体验的满意度对齐
- **对话系统**：用用户反馈替代对话质量自动评估
- **内容审核**：用编辑标注替代自动指标

### 混搭启发

**问：这项技术可以和哪些其他方法混搭？**

答：几种可能的混搭方向：

1. **+ 强化学习 (RLHF)**：DPO 本质是对比学习，可以与 RLHF 框架结合，让稀疏满意度信号作为 reward signal
2. **+ 大语言模型 (LLM)**：用 LLM 生成"模拟问卷反馈"来扩充稀疏数据，或用 LLM 做满意度推理
3. **+ 图神经网络 (GNN)**：将用户-内容交互图引入多任务学习，提供更好的协同信号
4. **+ 时序模型**：引入时序感知的满意度建模，区分"即时满意"和"长期满意"

### 反转启发

**问：如果反转这项技术的某个假设，会发生什么？**

答：一个有趣的反转方向：

**假设反转**：如果问卷信号不是"稀疏但准确"，而是"存在系统性偏差"会怎样？

- 例如：填写问卷的用户可能是极端满意或极端不满意的用户
- 反转后的研究：用因果推断（Causal Inference）方法，去除问卷的 selection bias
- 这可能延伸出另一个有价值的论文方向："因果增强的满意度对齐"
